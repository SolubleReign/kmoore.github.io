Enhancement Narratives

Software Engineering & Design

The Software Design and Engineering enhancement focused on refactoring the original monolithic game into a modular, object-oriented architecture. The project was reorganized into multiple files, including main.py, game.py, player.py, enemy.py, and room.py, separating responsibilities and improving maintainability. I selected this artifact because it demonstrates my growth in writing clean, maintainable, and scalable software, which is essential for real-world development. Key improvements include improved class design, separation of game logic, elimination of duplicated code, and clearer function responsibilities. These changes made the codebase significantly easier to extend for future features such as enemy tracking and database integration. Through this enhancement, I learned that functional code is not necessarily well-designed code. One of the biggest challenges was untangling tightly coupled logic from the original version. Many functions depended on shared global state, which required careful restructuring to preserve functionality while improving design. I also gained a deeper understanding of Python module imports and object interactions. This enhancement helped me meet course outcomes related to: • Developing modular and maintainable software systems • Applying object-oriented programming principles • Designing software solutions that align with requirements

Data Structures & Algorithms

The Algorithms and Data Structures enhancement introduced an Enemy Journal system, which tracks defeated enemies and stores their attributes such as name, health, and attack power. The system uses structured collections (dictionaries and lists) to store, search, and sort enemy data, adding a progression-based layer to gameplay. I selected this artifact because it demonstrates my ability to apply data structures and algorithms to solve meaningful in-game tracking problems. Instead of enemies being temporary encounters, they are now recorded and analyzed. Key skills demonstrated include selecting appropriate data structures, implementing search functionality by enemy name, and sorting enemies based on attributes. These improvements transformed the game from a simple combat loop into a structured system that tracks player progress over time. A major challenge was deciding how to structure enemy data efficiently. I tested multiple approaches before choosing a combination of dictionaries for fast lookups and lists for ordered storage. I also had to ensure consistency in how enemy attributes were stored to avoid errors in sorting and comparison logic. Feedback from earlier versions encouraged me to make the system more interactive and useful, which led to adding search and sorting features rather than only logging data. This enhancement helped me meet course outcomes related to: • Applying data structures to organize and process information • Designing algorithms for search and sorting • Extending program functionality using structured data

Databases

The Database enhancement upgraded the game from temporary in-memory storage to a persistent system using SQLite. This allowed player stats, inventory items, defeated enemies, and game progress to be saved and loaded between sessions. I included this artifact because it demonstrates my ability to implement persistent data storage and integrate databases into application logic, which is a key skill in modern software development. The enhancement significantly improved the user experience by allowing long-term progression rather than restarting each session. Key improvements include database schema creation, SQL queries for inserting and retrieving data, and integration of save/load functionality into gameplay flow. This required careful structuring of data to ensure consistency between the game state and stored records. One of the biggest challenges was managing synchronization between in-memory game data and the database. I had to carefully determine when to write updates to avoid duplication or data loss. I also gained experience debugging SQL queries and managing database connections effectively. Through this process, I learned how real applications separate logic from persistence layers and how important structured data storage is for scalable systems. This enhancement helped me meet course outcomes related to: • Designing and implementing database-driven applications • Integrating persistent storage into software systems • Managing structured data across application layers
